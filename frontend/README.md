### Silent refresh (Axios)

При ответе **401** клиент сначала вызывает `POST /auth/refresh` с `withCredentials: true` (cookie с refresh-токеном), сохраняет новый `accessToken` в Pinia (`applyAccessToken`) и **повторяет** исходный запрос. Разлогин и редирект на `/login` только если refresh не удался или 401 пришёл уже на refresh.

Чтобы refresh работал в браузере при разных origin у фронта и API, на бэкенде нужны `enableCors({ origin: …, credentials: true })` и согласованные настройки cookie (см. блок «План по бэкенду»).

## Идеи для фронтенда (бэклог)

Идеи ниже — ориентиры для UX и фич; реализовать можно выборочно.

### Продукт и контент

- **Сериалы по сезонам и сериям** — прогресс «где остановился», а не только фильм целиком; меньше трения при постоянном просмотре сериалов.
- **Свои списки и метки** — не только избранное: например «в дорогу», «с другом», «осень 2026»; опционально шаринг списка друзьям рядом с уже существующим чатом.
- **Быстрый ввод** — вставка ссылки или названия с автодополнением из каталога, чтобы быстрее вести заметки о просмотре.
- **Аналитика для пользователя** — осмысленные сводки: фильмов за месяц, любимые жанры, пересечения с друзьями (мотивация возвращаться, без «графиков ради графиков»).

### Онбординг и пустые состояния

- Короткий **онбординг** для новых пользователей: зачем приложение, как добавить первый тайтл, как пригласить друга.
- **Пустые состояния** с явным следующим шагом, чтобы не терять людей на пустом экране.

### Социальное и уведомления

- **In-app уведомления** (колокольчик): новые сообщения, заявки в друзья, ответы на рекомендации — без перезагрузки страницы; при желании позже — push в браузере.

### Доверие и доступность

- **Экспорт данных** (например JSON/CSV коллекции) — переносимость и ощущение контроля над своими данными.
- **Доступность**: фокус в модалках, контраст, уважение к `prefers-reduced-motion`.

### Технологии и удобство доступа

- **PWA / установка на домашний экран** — удобный доступ к «журналу просмотров», при желании офлайн-чтение списка.

### Внешние рейтинги

- Внешние рейтинги (IMDb / Кинопоиск) **не планируем**: нестабильные источники и лишняя сложность; при необходимости можно вернуться к обсуждению позже.

## Бэкенд (сделано в репозитории)

- **Swagger:** в `include` добавлены **Friendship**, **Message**, **Badge**; защищённые ручки помечены декоратором **`AuthProtected()`** (`ApiBearerAuth` + JWT). У **Users** и **Actors** добавлены `@ApiTags` / `@ApiOperation` где не хватало.
- **Роли:** в таблице `users` поле **`role`** (`USER` | `ADMIN`). **Каталог** (фильмы/актёры): мутации доступны **любому залогиненному пользователю** + лимит запросов (`AuthCatalogWrite`, анти-спам). **ADMIN** используется для администрирования **пользователей** (список всех, создание/удаление учёток и т.п.), а не для добавления фильмов в каталог.
- **JWT `@me`:** в `req.user` не попадает `password`.
- **Rate limit:** `ThrottlerModule` + лимиты на `login` / `register` / `refresh`; `logout` и `GET auth/@me` без лимита (`SkipThrottle`).
- **CORS** с `credentials` настраивается в `backend/src/main.ts` (`CORS_ORIGINS` / `FRONTEND_ORIGIN`).

### Что сделать локально после `git pull`

1. Миграции: `cd backend && npx prisma migrate deploy` (или `migrate dev` в разработке).
2. По желанию назначить **ADMIN** в БД (`role = 'ADMIN'`) — для модерации пользователей через админские ручки `/users`, а не для добавления фильмов.
3. В проде проверить **cookie** refresh (`COOKIE_DOMAIN`, `sameSite`, `secure`) под домен и HTTPS.
