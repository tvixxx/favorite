## Идеи для фронтенда (бэклог)

Идеи ниже — ориентиры для UX и фич; реализовать можно выборочно.

### Онбординг и пустые состояния

- Короткий **онбординг** для новых пользователей: зачем приложение, как добавить первый тайтл, как пригласить друга.
- **Пустые состояния** с явным следующим шагом, чтобы не терять людей на пустом экране.

### Доверие и доступность

- **Экспорт данных** (например JSON/CSV коллекции) — переносимость и ощущение контроля над своими данными.
- **Доступность**: фокус в модалках, контраст, уважение к `prefers-reduced-motion`.

### Технологии и удобство доступа

- **PWA / установка на домашний экран** — удобный доступ к «журналу просмотров», при желании офлайн-чтение списка.

### Внешние рейтинги

- Внешние рейтинги (IMDb / Кинопоиск) **не планируем**: нестабильные источники и лишняя сложность; при необходимости
  можно вернуться к обсуждению позже.

## Продукт и контент (статус)

Финальная сверка по четырём опорам из раздела «Продукт и контент» выше.

1. **Сериалы по сезонам и сериям — готово.** Бэкенд: нормализация прогресса и статуса в `UserMovieService`, поля
   сезона/серии.
   UI: `MovieDetail.vue` (сегмент статуса, ±сезон/серия, пресеты). Искать коллизии статуса при максимальном прогрессе не
   нужно — явный выбор пользователя сохраняется.

2. **Свои списки и метки — готово (MVP).** API `user-list`, Pinia `userListsStore`. UI: страница **«Мои списки»**
   (`MyListsPage.vue`: фильтры, метки, URL, удаление списка и тайтла из списка), модалка «В список» в `MovieDetail.vue`.
   **Не входит в MVP:** шаринг списка друзьям как отдельная фича (остаётся в идеях выше).

3. **Быстрый ввод — готово.** На странице коллекции (`MovieList.vue`) плавающая кнопка и панель с автодополнением по
   запросу
   к каталогу (`/movies/search`). **Уточнение:** удобнее всего вводить **название**; распознавание вставленной **только
   **
   URL-карточки `/detail/:id` в поле быстрого вода отдельно не разбирается (при желании — отдельная маленькая задача).

4. **Аналитика пользователя — готово (личная сводка).** Эндпоинт аналитики по коллекции, блок «Персональная аналитика»
   на
   `ProfilePage.vue` (тренд 7/30, топ жанров, продолжить смотреть, быстрые переходы в коллекцию с фильтрами).
   **Не сделано:** пересечения коллекции с друзьями — это социальная аналитика и другой объём данных/API (остаётся в
   бэклоге идей).

## Бэкенд (сделано в репозитории)

Подумать где использовать и с чем SSE (server side events).

- **Swagger:** в `include` добавлены **Friendship**, **Message**, **Badge**; защищённые ручки помечены декоратором \* \*
  `AuthProtected()`** (`ApiBearerAuth` + JWT). У **Users** и **Actors\*\* добавлены `@ApiTags` / `@ApiOperation` где не
  хватало.
- **Роли:** в таблице `users` поле **`role`** (`USER` | `ADMIN`). **Каталог** (фильмы/актёры): мутации доступны **любому
  залогиненному пользователю** + лимит запросов (`AuthCatalogWrite`, анти-спам). **ADMIN** используется для
  администрирования **пользователей** (список всех, создание/удаление учёток и т.п.), а не для добавления фильмов в
  каталог.
- **JWT `@me`:** в `req.user` не попадает `password`.
- **Rate limit:** `ThrottlerModule` + лимиты на `login` / `register` / `refresh`; `logout` и `GET auth/@me` без лимита (
  `SkipThrottle`).
- **CORS** с `credentials` настраивается в `backend/src/main.ts` (`CORS_ORIGINS` / `FRONTEND_ORIGIN`).

### Что сделать локально после `git pull`

1. Миграции: `cd backend && npx prisma migrate deploy` (или `migrate dev` в разработке).
2. По желанию назначить **ADMIN** в БД (`role = 'ADMIN'`) — для модерации пользователей через админские ручки `/users`,
   а не для добавления фильмов.
3. В проде проверить **cookie** refresh (`COOKIE_DOMAIN`, `sameSite`, `secure`) под домен и HTTPS.
